# 떡상농장 웹 프로토타입 v0.5

SNS에서 아직 뜨지 않은 영상 신호를 발견하고, 제한된 4개 슬롯에 심은 뒤, 가상 조회수와 다른 발견자들의 변화에 따라 수확하거나 갈아타는 핵심 루프를 테스트합니다.

## 바로 실행

YouTube 메타데이터 연결을 사용하려면 이 폴더에서 로컬 서버를 실행합니다.

Node.js 22.15 이상이 필요합니다.

```powershell
npm start
```

그다음 `http://127.0.0.1:4173`을 엽니다.

API 키가 없어도 YouTube oEmbed를 통해 실제 제목·채널·썸네일을 불러옵니다. 분석 시점의 실제 조회수와 업로드 시점까지 사용하려면 `.env.example`을 참고해 `YOUTUBE_API_KEY` 환경 변수를 설정한 뒤 서버를 실행합니다. PowerShell에서는 다음처럼 설정할 수 있으며 API 키는 브라우저에 노출되지 않습니다.

```powershell
$env:YOUTUBE_API_KEY="your-key"
npm start
```

전체 로직 검증은 다음 명령으로 실행합니다.

```powershell
npm run check
```

## 현재 구현된 것

- YouTube watch·Shorts·youtu.be·embed·live 링크 정규화
- YouTube의 실제 제목·채널·썸네일 조회
- 선택 사항인 YouTube Data API 키를 통한 실제 조회수·업로드 시점 조회
- 실제 영상 썸네일을 내 밭의 농작물 비주얼로 사용하는 `THUMBNAIL CROP`
- 12개 모의 영상이 경쟁하는 발견 피드
- 신호순·최신·저발견 정렬과 지켜보기
- 피드와 직접 입력 링크를 함께 저장하는 무제한 후보 보관함
- 활성 포지션·수확 완료 링크의 후보 재등록 차단
- 보관한 순간부터 계속 변하는 조회수·발견자·조기 발견 배수
- 후보별 `모든 변화`·`급등만`·`알림 끄기` 설정
- 새 후보와 현재 밭 4개를 한 화면에서 비교하는 스카우트 보드
- 피드에서 바로 후보를 분석하고 심는 흐름
- 시간에 따라 증가하는 피드 조회수와 가상 발견자
- 내 포지션과 지켜보는 신호의 변화를 알려주는 `SCOUT WIRE`
- URL을 붙여넣거나 샘플 링크를 선택해 포지션 생성
- 링크마다 결정되는 6종류의 가상 성장 곡선
- 동시에 보유 가능한 4개 슬롯
- 밭이 찼을 때 기존 포지션 수확 후 갈아타기
- 씨앗값을 100% 반환하는 무손실 수확
- 같은 유저가 수확한 링크에 다시 진입하지 못하는 일회성 발견
- 심을 때 확정되는 발견 순번과 이후 유입된 발견자 수
- `+1H`, `+6H` 테스트 시계와 24시간 자동 수확
- 진입 조회수, 성장률, 조기 발견 배수, 예상 수확가
- 수확 등급과 공유 문구
- 1080×1350 PNG 수확 증명 카드 저장·공유
- 수확 당시 발견 기록을 모아보는 발견 도감
- 브라우저 로컬 저장 및 초기화

## 의도적으로 제외한 것

- 실제 Instagram·TikTok API
- 서버 데이터베이스에 저장되는 조회수 시계열
- 로그인과 서버 데이터베이스
- 썸네일을 대체할 확정 농작물 아트
- 유저 간 거래와 현금성 보상

YouTube API 키가 없으면 시작 조회수도 데모 값입니다. API 키 유무와 관계없이 발견 순번과 심은 뒤의 성장 곡선은 아직 데모로 동작합니다. 현재 버전은 실제 영상의 정체성과 썸네일을 게임 루프에 연결하면서 `피드 탐색 → 촉으로 선택 → 선점 → 경쟁 관찰 → 수확 증명`이 재미있는지 확인하기 위한 프로토타입입니다.

버전별 변경 내용은 [`CHANGELOG.md`](./CHANGELOG.md)와 앱 하단의 `변경 기록`에서 확인할 수 있습니다.
