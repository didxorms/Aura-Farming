# 변경 기록

이 문서는 떡상농장 웹 프로토타입의 플레이 규칙과 주요 인터페이스 변경을 버전별로 기록합니다.

## [0.7.0] - 2026-07-24

### 추가

- 익명 플레이어 세션과 서버 권위의 잔액·심기·발견 순번·수확 API
- SQLite 마이그레이션과 플레이어·영상·스냅샷·피드·포지션·수확·지갑 원장
- YouTube 검색 레인, 묶음 통계 수집, 신호 점수 계산, 24시간 자동 수확 워커
- 외부 프런트와 API 분리를 위한 런타임 API 주소, CORS, Bearer 세션
- 실제 게임 기록과 분리된 시간 가속·모의 성장곡선 실험실
- 컨테이너 및 영구 볼륨 기반 외부 서버 배포 설정

### 변경

- 잔액, 발견 순번, 수확 결과의 기준을 브라우저 로컬 상태에서 서버 DB로 이전
- 발견 피드를 고정 모의 목록에서 서버가 수집한 실제 YouTube 후보 목록으로 전환
- 모의 기능은 서버 연결 실패 시 폴백 또는 실험실에서만 사용

## [0.6.1] - 2026-07-24

### 수정

- 조회수 동기화와 달리 테스트 시계에서만 증가하던 포지션 보유 시간을 실제 경과 시간에 연결
- 앱을 닫거나 백그라운드에 둔 동안 흐른 시간을 다음 실행·화면 복귀 시 복구
- 기존 v0.6 YouTube 포지션의 최초 조회수 스냅샷에서 심은 시각과 누락된 보유 시간을 자동 복원
- 실제 시계로 24시간이 지난 포지션도 자동 수확되도록 처리

## [0.6.0] - 2026-07-24

### 추가

- 활성 YouTube 포지션의 실제 조회수를 최대 50개씩 일괄 조회하는 서버 API
- 심은 시점의 실제 조회수와 이후 조회수 스냅샷을 로컬에 기록
- 앱이 보이는 동안 5분 간격 자동 동기화와 수동 `실제 조회수 갱신`
- 내 밭에 실제 추적 여부와 마지막 동기화 시점을 표시하는 `YOUTUBE LIVE` 바

### 변경

- API 키가 연결된 YouTube 포지션은 가상 곡선 대신 실제 조회수로 성장률·그래프·수확 가치를 계산
- 기존 v0.5 YouTube 포지션도 첫 실제 동기화 시점부터 기준 조회수를 새로 기록
- YouTube Data API를 사용할 수 없으면 기존 oEmbed·데모 성장 방식 유지
- 저장 데이터 스키마와 공유 빌드 표기를 v0.6으로 갱신

## [0.5.0] - 2026-07-23

### 추가

- YouTube watch·Shorts·youtu.be·embed·live 링크에서 동일한 영상 ID 추출
- API 키 없이도 oEmbed로 실제 영상 제목·채널·썸네일 조회
- `YOUTUBE_API_KEY` 환경 변수가 있으면 실제 조회수와 업로드 시점 조회
- 서버 전용 YouTube 조회 경로로 API 키가 브라우저에 노출되지 않도록 구성
- 내 밭과 후보·포지션 상세 화면에 실제 썸네일을 농작물처럼 표시하는 `THUMBNAIL CROP`

### 변경

- YouTube URL 형식이 달라도 같은 영상을 하나의 후보·포지션으로 인식
- 저장 데이터 스키마를 v5로 올리고 v0.4 이하 데이터를 자동 이전
- 실제 데이터와 데모 조회수의 구분을 후보 상세 화면에 표시
- 공유 문구와 빌드 표기를 v0.5로 갱신

## [0.4.1] - 2026-07-23

### 수정

- 이미 내 밭에 심은 영상을 발견 피드에서 후보 보관함에 다시 넣을 수 있던 문제 수정
- 수확 완료한 영상도 후보 보관 버튼이 비활성화되도록 수정
- v0.4 저장 데이터에 활성·수확 영상이 후보로 남아 있으면 로딩 시 자동 제거

### 개선

- 후보와 내 밭을 가로로 압축한 표 대신 제목·상태·4개 지표가 분리된 카드로 비교
- 모바일 비교 지표를 2열로 배치해 긴 제목과 숫자가 잘리지 않도록 개선
- 밭이 가득 찼을 때 선택한 교체 카드의 테두리와 상태 문구를 강화
- 공유 문구와 빌드 표기를 v0.4.1로 갱신

## [0.4.0] - 2026-07-23

### 추가

- 발견 피드와 직접 입력 링크를 함께 보관하는 `SCOUT DESK`
- 개수 제한 없이 원하는 신호를 보관하고 나중에 심는 흐름
- 후보별 `모든 변화`·`급등만`·`알림 끄기` 설정
- 후보와 현재 밭의 조회수·신호·발견 순번을 한 화면에서 비교하는 스카우트 보드
- 수확 결과를 1080×1350 PNG 증명 카드로 저장하고 지원 환경에서 이미지로 공유하는 기능
- 발견 도감의 기록을 다시 열어 결과 카드로 내보내는 흐름

### 변경

- 피드 전용 `지켜보기`를 임의 링크에도 사용할 수 있는 영속 후보 데이터로 확장
- 기다리는 동안 후보의 조회수와 발견자가 계속 증가하고 조기 발견 배수가 낮아지도록 변경
- 밭이 가득 찼을 때 별도 교체 목록 대신 비교 보드에서 수확할 포지션을 직접 선택
- 공유 문구와 빌드 표기를 v0.4로 갱신

### 데이터

- v0.3의 지켜보기 목록을 v0.4 후보 보관함으로 자동 마이그레이션
- v0.1·v0.2·v0.3 저장 데이터를 유지하면서 후보별 알림 설정 추가

## [0.3.0] - 2026-07-23

### 추가

- 12개의 모의 숏폼 신호가 함께 움직이는 발견 피드
- 신호순·최신·저발견 피드 정렬
- 후보를 보류하고 변화를 추적하는 지켜보기
- 피드에서 현재 조회수와 발견자 수를 기준으로 바로 진입하는 흐름
- 내 포지션의 후발 발견자와 지켜보는 신호의 움직임을 전하는 `SCOUT WIRE`
- 수확 당시 진입 조회수·발견 순번·성장 결과를 모아보는 발견 도감
- 하단 내비게이션을 통한 내 밭·발견·발견 도감 화면 전환

### 변경

- 테스트 시계를 넘기면 내 밭뿐 아니라 발견 피드 전체도 함께 진행
- 피드에서 심을 때 그 순간의 조회수와 발견자 수를 진입 기록으로 고정
- 결과 공유 문구와 빌드 표기를 v0.3으로 갱신

### 데이터

- v0.1·v0.2 브라우저 저장 데이터를 v0.3 형식으로 자동 마이그레이션
- 기존 포지션과 수확 기록을 유지하면서 지켜보기·소식·현재 화면 상태 추가

## [0.2.0] - 2026-07-23

### 추가

- 링크를 심는 순간 확정되는 발견 순번
- 시간이 흐르며 증가하는 전체 발견자 수와 `내 뒤로 들어온 발견자` 표시
- 후보 분석, 밭 카드, 포지션 상세, 수확 결과에 발견자 기록 연결
- 앱 안에서 확인할 수 있는 버전별 변경 기록

### 변경

- 모든 수확에서 씨앗값 1,000코인을 100% 반환하도록 보상 하한 변경
- 성장한 조회수에 대해서만 추가 보너스를 지급하는 무손실 구조로 전환
- 교체 후 즉시 새 링크를 심을 수 있도록 무쿨타임 유지
- 수확 결과 공유 문구를 수익보다 발견 순번 중심으로 변경

### 제한

- 한 번 수확한 링크는 같은 유저가 다시 심을 수 없음
- 이미 밭에 있는 링크는 중복으로 심을 수 없음

### 데이터

- v0.1 브라우저 저장 데이터를 v0.2 형식으로 자동 마이그레이션
- v0.1에서 씨앗값보다 적게 수확한 기록이 있다면 차액을 자동 환급

## [0.1.0] - 2026-07-23

### 추가

- URL 붙여넣기와 샘플 링크 분석
- 링크별 6종류의 가상 조회수 성장 곡선
- 동시에 보유 가능한 4개 포지션 슬롯
- 기존 포지션 수확 후 새 링크로 교체하는 선택 화면
- `+1H`, `+6H` 테스트 시계와 24시간 자동 수확
- 진입 조회수, 성장률, 조기 발견 배수, 예상 수확가
- 수확 등급과 결과 공유 문구
- 브라우저 로컬 저장과 초기화
